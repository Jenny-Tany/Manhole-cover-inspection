# 生成式AI教育平台项目规划

## 核心功能
- 课程管理系统
  - 教师创建、管理课程
  - 学生浏览、选择课程
- 知识库功能
  - 相关领域知识库供AI模型使用
  - 学生可以与AI互动
  - 基于知识库提供解题详细步骤
- 个性化学习路径
  - 基于学习进度、考试成绩和AI推荐生成学习路径
  - 帮助学生更好掌握学习内容

## AI与学生的交互方式
- 文本对话
  - 学生输入问题
  - AI返回解题步骤和知识点
- 基于通义千问模型的问答与解题支持

## 技术栈
- **前端**: Vue3 + TypeScript
  - 界面开发
  - 课程管理与学习路径展示
  - 与AI的文本交互
- **后端**: Spring生态系统
  - Spring Boot：基础服务开发
  - Spring Cloud：微服务架构支持
  - 数据存储与管理（MySQL、MongoDB等）
- **AI模型**: 通义千问
  - 问答与解题能力
  - 连接知识库，提供个性化解答
- **登录方式**
  - Google账号登录
  - 手机号登录
  - 微信扫码登录

## 语言支持
- 平台仅支持中文

## 可扩展性与安全性
- 高度扩展性
  - 支持未来功能扩展
  - 动态知识库更新
- 安全性
  - 用户数据保护
  - 访问权限管理

## 建议的技术架构
- **前端**
  - Vue3 + TypeScript
  - Axios用于API请求
  - Vuex用于状态管理
- **后端**
  - Spring Boot + Spring Cloud构建微服务架构
  - Spring Security保障平台安全
  - MySQL/MongoDB用于存储课程、用户信息、学习路径
- **AI模型**
  - 使用通义千问模型处理问答与解题任务
  - 知识库通过后端API动态更新
- **API网关**
  - 使用Spring Cloud Gateway
  - 支持微服务路由与API访问控制
- **身份验证**
  - 使用OAuth 2.0 + JWT实现Google登录
  - 微信开放平台接口支持扫码登录
  - 基于手机号的多因素身份验证

## 潜在挑战
- AI模型的知识库管理
  - 动态更新领域知识的机制
  - 知识库数据如何精准应用于AI解题
- 学生个性化学习路径生成
  - 如何准确跟踪学习进度
  - 考试成绩和AI推荐之间的平衡
- 平台扩展性
  - 微服务架构在功能增加时的性能优化
  - 大量用户请求对AI模型性能的影响
- 安全与隐私
  - 数据库中的敏感信息如何加密存储
  - API访问权限与用户数据保护
